# avatar-gen

Vibe-coded slop for generating variations of my GitHub profile picture. Tweaked until I didn't like it anymore so I've stopped. Got a bit out of hand.

Generates layered SVGs with animated color waves, an organic multi-petal flower, and a polygon node ring. Can export to PNG/JPEG and assemble animated GIFs.

## Example Output

Animated output from the `green_blue_galaxy` large preset:

![Green-Blue Galaxy large animated output](examples/green-blue-galaxy-large.gif)

## Requirements

- Node.js (developed on v25)
- `npm install`
- For PNG/JPEG export: Chromium (set `CHROMIUM_EXEC` env var or have it on `PATH`)
- For GIF assembly: ImageMagick (`convert` / `magick`)

## Quick start

```bash
npm install
npm run pull-fonts        # install @fontsource packages listed in config/fonts.json

# Generate a single SVG using a preset
npm run generate -- --json presets/green_blue_galaxy/single-svg-output.twinkle-strength-1.generate.json

# Generate and convert to PNG/JPEG
npm run generate -- --json presets/green_blue_galaxy/single-svg-output.twinkle-strength-1.generate.json \
  | sed -n 's/^Done: //p' \
  | xargs -r -I {} npm run convert -- --json presets/green_blue_galaxy/single-svg-and-converted-outputs.convert.json --input-files {}

# Generate an animated GIF (small/quick)
npm run generate-animated -- --json presets/green_blue_galaxy/animated-gif-output.small-and-quick.animate.json

# Generate an animated GIF (production quality)
npm run generate-animated -- --json presets/green_blue_galaxy/animated-gif-output.large.animate.json
```

## JSON-based configuration

Every script takes a single `--json <path>` argument pointing to a config file validated
against its corresponding JSON Schema. This replaces the old per-flag CLI interface.

Keys use **camelCase** (`waveCount`, `flowerSeed`, `twinkleStrength`, etc.).

The schemas live in `schemas/` and are regenerated by `npm run generate-schemas`. VS Code
auto-associates them via `.vscode/settings.json` so you get autocompletion and validation
in-editor.

## Commands

### `npm run generate`

Generates a single SVG avatar.

```bash
npm run generate -- --json <path/to/preset.generate.json>
npm run generate -- --help
```

| Field | Description |

| `flowerSeed` | Independent RNG seed for `feTurbulence` flower shape |
| `waveCount` | Number of color waves |
| `waveStartDegree` | Starting hue (oklch degrees) |
| `waveDistance` | Total hue sweep topâ†’bottom; negative sweeps backward |
| `waveLightness` / `waveChroma` | Wave oklch L / C |

| `lobeCount` | Number of flower petals |
| `lobeBumpiness` | Petal edge roughness (scalar or array per layer) |

### `npm run generate-animated`

Renders one SVG frame per animation step, then calls ImageMagick to assemble a GIF.

```bash
npm run generate-animated -- --json <path/to/preset.animate.json>
```

Animation-specific fields (in addition to all generate fields):

| Field | Default | Description |
|---|---|---|
| `seeds` | auto | Array of per-cycle RNG seeds |
| `flowerSeed` | `seeds[0]` | Fixed turbulence seed (held constant across cycles) |
| `fps` | 10 | Frames per second |
| `timePerTwinkleMs` | 1000 | Single value or four-element array: `[rise, hold-high, fall, hold-low]` |
| `twinkleCycles` | 4 | Number of unique seed cycles to render |
| `res` | 400 | Output resolution in pixels (square) |
| `name` | `"generated_animation"` | Base name for the output GIF |

Keeping `flowerSeed` fixed across cycles prevents the flower from reshaping between loops.

### `npm run convert`

Converts SVG(s) to PNG (transparent) and JPEG (white background) at multiple resolutions
using Puppeteer/Chromium. Handles `oklch()` colors and embedded `@fontsource` fonts correctly.

```bash
npm run convert -- --json <path/to/preset.convert.json>
```

| Field | Description |
|---|---|
| `inputFiles` | Array of SVG file paths to convert |
| `outputDir` | Parent directory for output sub-directories |
| `textFont` | Override font family (auto-detected from SVG if omitted) |

Output resolutions: 200, 400, 600, 800, 1000, 1080, 1200, 1600, 2000, 4000 px.

### `npm run generate-cycle-fonts`

Renders one avatar per font in `config/fonts.json`, all other params identical, for easy
font comparison. Output goes to a timestamped sub-directory.

```bash
npm run generate-cycle-fonts -- --json <path/to/preset.generate.json>
```

### `npm run pull-fonts`

Installs the `@fontsource` packages listed in `config/fonts.json` as optional dependencies.
Already-installed fonts are skipped.

### `npm run generate-schemas`

Regenerates `schemas/*.schema.json` from the Zod schemas in `src/schemas.js`, and
overwrites `presets/default/` with fresh skeleton files containing all current defaults.

### `npm run modify-templates`

Bulk-patches one or more config JSON files, with a git-style character-level colored diff
shown for every change applied. Accepts files or directories (searched recursively).

```bash
# Patch all presets in a directory
npm run modify-templates -- presets/green_blue_galaxy --waveCount 8 --waveLightness 0.3

# Patch a single file
npm run modify-templates -- presets/green_blue_galaxy/animated-gif-output.large.animate.json --twinkleCycles 5

# Patch files in multiple targets at once
npm run modify-templates -- presets/green_blue_galaxy presets/default --seed 12345

# Values can be any JSON type: numbers, booleans, arrays, strings
npm run modify-templates -- presets/ --flowerColors '["oklch(0.85 0.18 171)","oklch(0.7 0.15 200)"]'
```

Recognized file suffixes: `*.generate.json`, `*.animate.json`, `*.convert.json`.
Unchanged files are listed but not written.

## Configuration

### `config/fonts.json`

Array of `{ "name": "...", "package": "@fontsource/..." }` entries. Edit this to add or
remove fonts available to `textFont` and `generate-cycle-fonts`.

## Notes

- The SVG is 400Ă—400 px internally; use `npm run convert` to export at other sizes.
- `seed` controls polygon node placement and twinkle selection. `flowerSeed` independently
  controls `feTurbulence` organic wobble â€” keeping it fixed across animation cycles prevents
  the flower from reshaping between loops.
- `lobeBumpiness` and `flowerTurbulenceFreq` accept a single value or an array applied per
  layer from innermost to outermost (cycling if fewer values than layers).
- `lobeGapExponent` > 1 compresses inner gaps, creating a tunnel-depth effect.
- Colors throughout use oklch color space: `oklch(lightness chroma hue)` â€” which maps
  naturally to the continuous hue sweep the wave system generates.
- When passing negative numbers as flag values (e.g. `--waveDistance -135`), use the
  `--key=value` form to avoid shell ambiguity: `--waveDistance=-135`.
